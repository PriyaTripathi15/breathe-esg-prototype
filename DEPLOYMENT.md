# Deployment

This repository is wired for a free split deployment:

- `backend/` on Render as a web service
- `frontend/` on Vercel as a static site
- Postgres from Supabase for persistent storage

## Services

- `backend/` deploys as a Python web service on Render.
- `frontend/` deploys as a static site on Vercel.
- Supabase Postgres backs the Django API in production.

## Production Environment Variables

Backend:
- `DJANGO_SECRET_KEY`: required, generated by Render or set manually.
- `DJANGO_DEBUG`: set to `false` in production.
- `DJANGO_ALLOWED_HOSTS`: the backend service hostname, for example `breathe-esg-api.onrender.com`.
- `DJANGO_CORS_ALLOWED_ORIGINS`: the frontend URL, for example `https://your-app.vercel.app`.
- `DATABASE_URL`: provided by Supabase.

Frontend:
- `VITE_API_BASE_URL`: the backend API URL, for example `https://breathe-esg-api.onrender.com/api`.

## Free Deployment Flow

1. Create a free Supabase project and copy the Postgres connection string.
2. Create a Render web service from `backend/`.
3. Set the backend environment variables listed above, including `DATABASE_URL`.
4. Create a Vercel project from `frontend/`.
5. Set `VITE_API_BASE_URL` in Vercel to your Render backend URL ending in `/api`.
6. Deploy both services.

## Local Run

Backend:
- `pip install -r requirements.txt`
- `python manage.py migrate`
- `python manage.py runserver`

Frontend:
- `npm install`
- `npm run dev`

## Backend Procfile

The backend includes a `Procfile` for process-based deploy targets:

```procfile
web: gunicorn breathe_esg.wsgi:application
```
