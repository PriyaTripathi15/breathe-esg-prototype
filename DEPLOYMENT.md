# Deployment

This repository is wired for Render.

## Services

- `backend/` deploys as a Python web service.
- `frontend/` deploys as a static site.
- A Render Postgres database backs the Django API in production.

## Production Environment Variables

Backend:
- `DJANGO_SECRET_KEY`: required, generated by Render or set manually.
- `DJANGO_DEBUG`: set to `false` in production.
- `DJANGO_ALLOWED_HOSTS`: the backend service hostname, for example `breathe-esg-api.onrender.com`.
- `DJANGO_CORS_ALLOWED_ORIGINS`: the frontend URL, for example `https://breathe-esg-web.onrender.com`.
- `DATABASE_URL`: provided by the attached Postgres database.

Frontend:
- `VITE_API_BASE_URL`: the backend API URL, for example `https://breathe-esg-api.onrender.com/api`.

## Seeded Demo Login

The backend seeds demo credentials automatically on migrate.

- `analyst@breatheesg.com` / `BreatheESG2026!`
- `auditor@breatheesg.com` / `BreatheESG2026!`

## Local Run

Backend:
- `pip install -r requirements.txt`
- `python manage.py migrate`
- `python manage.py runserver`

Frontend:
- `npm install`
- `npm run dev`
